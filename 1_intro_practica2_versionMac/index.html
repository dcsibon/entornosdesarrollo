Práctica 2 – Del código fuente al ejecutable (con IDE) - versióm macOS

Parte 1. Código inicial con errores

  1. Copia este programa en un archivo llamado bucle2.c (contiene 2 errores de sintaxis intencionados):
#include <stdio.h>
#include <time.h>

int main() {
    clock_t inicio, fin
    inicio = clock();

    long long suma = 0
    for (long long i = 0; i < 100000000; i++) {
        suma += i;
    }

    fin = clock();
    double tiempo = (double)(fin - inicio) / CLOCKS_PER_SEC;

    printf("Suma final: %lld\n", suma);
    printf("Tiempo en segundos: %f\n", tiempo);
    return 0;
}
  1. Intenta compilar desde la Terminal:

bash gcc bucle2.c -o bucle2

  1. Anota el mensaje de error que aparece.

Parte 2. Uso de un IDE

  1. Instala Visual Studio Code (con Homebrew)

bash brew install --cask visual-studio-code

  1. Abre Visual Studio Code y entra en la sección de Extensiones (icono de bloques en la barra lateral izquierda o Ctrl+Shift+X).

  2. Instala las siguientes extensiones de Microsoft (como se ven en la captura):

  3. C/C++ → soporte para IntelliSense, resaltado de errores y depuración.

  4. C/C++ Extension Pack → incluye configuraciones útiles adicionales.

  5. (Opcional) C/C++ Themes → cambia solo la apariencia de la extensión.

  6. Selecciona el compilador por defecto (solo la primera vez):

  7. Al abrir un .c, puede salir un aviso para elegir compilador: selecciona /usr/bin/gcc (o /usr/bin/clang; ambos valen).

  8. Si no aparece, ve a View → Command Palette… (⌘⇧P) → C/C++: Edit Configurations (UI) → en Compiler path elige /usr/bin/gcc.

  9. Abre el archivo bucle2.c en VS Code.

  10. Corrige los 2 errores de sintaxis. El propio VS Code te marcará con subrayado rojo dónde están los fallos. Al posicionarse encima con el ratón muestra el mensaje de error de sintaxis.

  11. Compila y ejecuta desde el mismo IDE con Ctrl+F5. Aunque también, desde la pestaña inferior Terminal que tiene integrada, podéis lanzar, una vez compilado, directamente el programa escribiendo el comando .\bucle2.

  12. Haz una captura de pantalla mostrando la ejecución correcta, con la suma final y el tiempo medido.
