    U3 - Control de versiones (Git + GitHub)

    INTRODUCCIÓN

    En el desarrollo profesional de software, es fundamental dominar sistemas de control de versiones que permitan gestionar el código fuente de forma segura, colaborativa y eficiente. Git es la herramienta más extendida para este propósito, y GitHub es la plataforma más popular para alojar repositorios en la nube, facilitando la colaboración y sincronización entre desarrolladores.

    En este tema aprenderemos a trabajar con Git en local, conectarlo con GitHub, sincronizar cambios, gestionar ramas y controlar el historial de versiones.

    OBJETIVOS

    • Comprender los fundamentos del control de versiones distribuido con Git.
    • Crear y gestionar repositorios locales.
    • Configurar la conexión con GitHub mediante HTTPS o SSH.
    • Sincronizar cambios entre el repositorio local y remoto (push, pull, fetch, clone, fork).
    • Trabajar de forma estructurada con ramas y fusiones (merge).
    • Consultar, revertir o restaurar versiones anteriores mediante el historial de commits.
    • Aplicar buenas prácticas en el trabajo colaborativo y en la gestión del código.

    ÍNDICE DE LA UNIDAD

    Nº Apartado Contenido principal
    1 Git básico (trabajo en local) Flujo local, init, add, commit, status, log
    2 Conexión con GitHub Tipos de conexión: HTTPS, SSH, token, credential helper
    3 Sincronización con repos remotos push, pull, fetch, clone, fork, remote
    4 Gestión de ramas Creación, fusión, conflictos, eliminación y buenas prácticas
    5 Gestión del historial de commits Consulta, revert, reset, checkout y navegación entre versiones
    6 Hojas resumen del tema (chuletas) Síntesis de comandos y flujos visuales
